Tipos de datos:
tipos primitivos: sin definir (undefined), nulo (null),lógicos (boolean), numérico(number), cadena (string) y símbolo(symbol):
Sin definir (undefined): la variable no definida tiene como valor este tipo de dato, no es un objeto y se puede identificar con, typeof o myVar=== undefined ya que hay una variable con el nombre undefined.
Nulo (null):Es un objeto según typeoff pero no es una instancia de object, hay que dar un tratamiento especifico de null, hay una única forma de comprobar si el valor es nulo y es compararlo con el literal null.
Simbolo(symbol): se usan como claves en objetos y mapas de datos, no se pueden convertir a otros tipos, por lo que, para acceder a ellos, tendremos que hacer referencias al símbolo con el que fue creado. Cuando se crea un símbolo se obtien.e un valor único por eso son de utilidad para crear constantes con valores únicos y no producir confusiones para valores complicados, la mejor forma de saber si un valor es de tipo symbol es utilizar typeoff.

Logico(boolean):es un objeto contenedor parqa un valor booleano, el valor, como primer parámetro si es necesario se convierte en valor booleano, si el valor se omite o es 0,-0,null, false, NaN, undefined o “ “ (casillas vacias) el valor inicial del objeto es false, todos los demás valores, incuido cualquier objeto, la cadena “false” o un arreglo vacio, tendrá un valor inicial de true, no hay que confundir los valores del boolean primitivo, true y false, con los del objeto boolean. Cualquier objeto cuyo valor sea undefined o null, un objeto boolean cuyo valor es false se evalua como true cuando se pasa a declaración condicional. Para convertir un valor no booleano en uno booleano, usa boolean como función. Si especificas un objeto boolean cuyo valor es false como valor inicial el nuevo objeto tiene valor de true, no se usa un objeto boolean en lugar de boolean primitivo.

Number: objeto primitivo emvolvente que permite representar y manipular valores numéricos, contiene constantes y métodos para trabajar con números y valores que pueden ser convertidos a números usando la función number (), los usos del objeto number (valor) son: convewrtir un string a un tipo numérico, si no puede ser convertido en numero, devuelve n a n.

Cadena (string): se usa para representar datos textuales, es un conjunto de elementos mde valores enteros si signos de 16 bits, cada elemento del string ocupa sua lugar en la cadena, la longitud de cadena depende de la cantidad de elementos que contenga, las cadenas son inmutables, es decir, que al crear una cadena, no la podremos modificar, pero es posible crear otra cadena basada en una operación en la cadena original.
Beneficios de usar cadena para representar datos complejos:

  • construir cadenas complejas y concatenadas es sencillo.
  • son fáciles de depurar.
  • las cadenas son el denominador común de muchas Apis.
