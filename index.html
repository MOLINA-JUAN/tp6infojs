Tipos de datos:
tipos primitivos: sin definir (undefined), nulo (null),lógicos (boolean), numérico(number), cadena (string) y símbolo(symbol):
Sin definir (undefined): la variable no definida tiene como valor este tipo de dato, no es un objeto y se puede identificar con, typeof o myVar=== undefined ya que hay una variable con el nombre undefined.
Nulo (null):Es un objeto según typeoff pero no es una instancia de object, hay que dar un tratamiento especifico de null, hay una única forma de comprobar si el valor es nulo y es compararlo con el literal null.
Simbolo(symbol): se usan como claves en objetos y mapas de datos, no se pueden convertir a otros tipos, por lo que, para acceder a ellos, tendremos que hacer referencias al símbolo con el que fue creado. Cuando se crea un símbolo se obtien.e un valor único por eso son de utilidad para crear constantes con valores únicos y no producir confusiones para valores complicados, la mejor forma de saber si un valor es de tipo symbol es utilizar typeoff.

Logico(boolean):es un objeto contenedor parqa un valor booleano, el valor, como primer parámetro si es necesario se convierte en valor booleano, si el valor se omite o es 0,-0,null, false, NaN, undefined o “ “ (casillas vacias) el valor inicial del objeto es false, todos los demás valores, incuido cualquier objeto, la cadena “false” o un arreglo vacio, tendrá un valor inicial de true, no hay que confundir los valores del boolean primitivo, true y false, con los del objeto boolean. Cualquier objeto cuyo valor sea undefined o null, un objeto boolean cuyo valor es false se evalua como true cuando se pasa a declaración condicional. Para convertir un valor no booleano en uno booleano, usa boolean como función. Si especificas un objeto boolean cuyo valor es false como valor inicial el nuevo objeto tiene valor de true, no se usa un objeto boolean en lugar de boolean primitivo.

Number: objeto primitivo emvolvente que permite representar y manipular valores numéricos, contiene constantes y métodos para trabajar con números y valores que pueden ser convertidos a números usando la función number (), los usos del objeto number (valor) son: convewrtir un string a un tipo numérico, si no puede ser convertido en numero, devuelve n a n.

Cadena (string): se usa para representar datos textuales, es un conjunto de elementos mde valores enteros si signos de 16 bits, cada elemento del string ocupa sua lugar en la cadena, la longitud de cadena depende de la cantidad de elementos que contenga, las cadenas son inmutables, es decir, que al crear una cadena, no la podremos modificar, pero es posible crear otra cadena basada en una operación en la cadena original.
Beneficios de usar cadena para representar datos complejos:

  • construir cadenas complejas y concatenadas es sencillo.
  • son fáciles de depurar.
  • las cadenas son el denominador común de muchas Apis.

6-Estructuras de control:
JS utiliza estructuras de control, estas son herramientas que permiten controlar el flujo de las operaciones del programa, un programa sin estructuras de control es inútil, las estructuras de control son:
Estructura If/else: If representa una parte del código que se ejecuta si una condición es verdadera, en caso de que esta condición sea falsa, esta estructura permite utilizar otra parte del código. Se usa de esta manera:
If (condicion){
Instrucción
}
El else no es obligatoria y se usa asi:
If (condicion){
intruccion
}else{
intruccion
}
-En el caso de tener varios else, que serian como tener rebotes en nuestro código, ya que la primera condición no se cumple y tampoco la segunda por ejemplo, podemos hacer esto:
if(condicion){
instruccion
}else if(condicion){
instruccion
}else if(condicion){
instruccion
}
Estructura Switch: utilizaremos esta estructura en caso de que tengamos más de una acción y diferentes condiciones, ya que se vuelve muy eficiente y para esto, vamos a darle valores y comprobaremos si el valor esta dentro del código de la estructura, la usaremos asi: switch(variable) {
case valor_1:
break;
case val or_2:
break;
case valor_n:
break;
default:
break;
}
La estructura switch se define mediante la palabra reservada switch seguida, entre paréntesis, del Nombre de la variable que se va a utilizar en las comparacione, las instrucciones que forman parte del switch se encierran entre las llaves { y }.En el swicht se van a definir las comparaciones que se requieren realizar sobre el valor de la variable, cada una de ellas se indica mediante la palabra reserva seguida del valor con la que se hace la comparación, si dicho valor coincide con el valor indicado por case se ejecutan las instrucciones definidas dentro de esa clase, luego de las instrucciones del case se incluye la sentencia breack para terminar el switch (no es obligatorio), las comparaciones se hacen por orden, siendo importante el orden en el que se definen los case, si el valor de switch no coincide con los valores de case, se usa el valor default, aunque es opcional, los switch los usan para definir cuando no es correcta la opción y mostrar un mensaje por pantalla.
Estructura while: Esta estructura permite crear bucles que se ejecutan ninguna o más veces, dependiendo de la condición indicada. Su funcionamiento de define: "mientras se cumpla la condición indicada, repite indefinidamente las instrucciones incluidas dentro del bucle", si esto no se cumple, el bucle no se ejecutara, por otro lado, si se hace, se ejecuta una vez y vuelve a comprobar esta condición, si esta se sigue cumpliendo se sigue ejecutando el bucle hasta que esta condición no se cumpla.
while(condicion) {
instruccion
actualizacion
}
Estructura for: esta estructura permite hacer peticiones al usuario de casos/sentencias o instrucciones de una forma sencilla, mientras la condición indicada en el for, se siga cumpliendo, se va a repetir la ejecución del código dentro del for, al terminar una vuelta/repetición se suma a las variables que se usaron en la condición. El for se usa asi:
For (inicialización; condición; actualizacion){
Instrucción
actualizacion
}
Inicialización: acá se establecen los valores donde comienza el bucle y variables que controlan las repeticiones.
Condición: esta elemento decide si el código se va a seguir repitiendo o no.
Actualización: el valor que se asigna a las variables que controlan el programa después de cada repetición.
Estructura do...while: esta estructura es muy parecida a la etructura while, lo que los hacen diferentes son, que en este bucle, el programa siempre se repite excepto la primera vez, esta estructura comprueba el programa cada vez que se repite, la primera vez siempre se ejecuta todas las instrucciones. En el while, siempre deben agregarse caracteres, esto es no se hace en el while común. Esta estructura se usa asi:
Do{
Instrucción
}while (condicion);

A través del DOM podemos acceder por medio de js a sus correspondientes objetos para alterar sus propiedades o invocar sus métodos, esta definido y administrado por el W3C y los distintos navegadores aplican las especificaciones del world wide web consortium, para darle soporte a sus aplicaciones, no solo permite modificar páginas web en html, sino también documentos xml. Los niveles de DOM cambian según el navegador y las especificaciones se entienden de distinta manera por las distintas organizaciones creadas por los navegadores, se creó un marco donde trabajar con los objetos difiere de un navegador a otro. En js, la forma de acceder al DOM es a través del objeto llamado document que representa al árbol de DOM.,en su interior pueden existir varios tipos de elementos que serán principalmente element o node.
