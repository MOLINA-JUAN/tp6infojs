Logico(boolean):es un objeto contenedor parqa un valor booleano, el valor, como primer parámetro si es necesario se convierte en valor booleano, si el valor se omite o es 0,-0,null, false, NaN, undefined o “ “ (casillas vacias) el valor inicial del objeto es false, todos los demás valores, incuido cualquier objeto, la cadena “false” o un arreglo vacio, tendrá un valor inicial de true, no hay que confundir los valores del boolean primitivo, true y false, con los del objeto boolean. Beneficios de usar cadena para representar datos complejos:

-Estructuras de control:
JS utiliza estructuras de control, estas son herramientas que permiten controlar el flujo de las operaciones del programa, un programa sin estructuras de control es inútil, las estructuras de control son:
Estructura If/else: If representa una parte del código que se ejecuta si una condición es verdadera, en caso de que esta condición sea falsa, esta estructura permite utilizar otra parte del código. Se usa de esta manera:
If (condicion){
Instrucción
}
El else no es obligatoria y se usa asi:
If (condicion){
intruccion
}else{
intruccion
}
-En el caso de tener varios else, que serian como tener rebotes en nuestro código, ya que la primera condición no se cumple y tampoco la segunda por ejemplo, podemos hacer esto:
if(condicion){
instruccion
}else if(condicion){
instruccion
}else if(condicion){
instruccion
}
Estructura Switch: utilizaremos esta estructura en caso de que tengamos más de una acción y diferentes condiciones, ya que se vuelve muy eficiente y para esto, vamos a darle valores y comprobaremos si el valor esta dentro del código de la estructura, la usaremos asi: switch(variable) {
case valor_1:
break;
case val or_2:
break;
case valor_n:
break;
default:
break;
}
La estructura switch se define mediante la palabra reservada switch seguida, entre paréntesis, del Nombre de la variable que se va a utilizar en las comparacione, las instrucciones que forman parte del switch se encierran entre las llaves { y }.En el swicht se van a definir las comparaciones que se requieren realizar sobre el valor de la variable, cada una de ellas se indica mediante la palabra reserva seguida del valor con la que se hace la comparación, si dicho valor coincide con el valor indicado por case se ejecutan las instrucciones definidas dentro de esa clase, luego de las instrucciones del case se incluye la sentencia breack para terminar el switch (no es obligatorio), las comparaciones se hacen por orden, siendo importante el orden en el que se definen los case, si el valor de switch no coincide con los valores de case, se usa el valor default, aunque es opcional, los switch los usan para definir cuando no es correcta la opción y mostrar un mensaje por pantalla.
Estructura while: Esta estructura permite crear bucles que se ejecutan ninguna o más veces, dependiendo de la condición indicada. Su funcionamiento de define: "mientras se cumpla la condición indicada, repite indefinidamente las instrucciones incluidas dentro del bucle", si esto no se cumple, el bucle no se ejecutara, por otro lado, si se hace, se ejecuta una vez y vuelve a comprobar esta condición, si esta se sigue cumpliendo se sigue ejecutando el bucle hasta que esta condición no se cumpla.
while(condicion) {
instruccion
actualizacion
}
Estructura for: esta estructura permite hacer peticiones al usuario de casos/sentencias o instrucciones de una forma sencilla, mientras la condición indicada en el for, se siga cumpliendo, se va a repetir la ejecución del código dentro del for, al terminar una vuelta/repetición se suma a las variables que se usaron en la condición. El for se usa asi:
For (inicialización; condición; actualizacion){
Instrucción
actualizacion
}
Inicialización: acá se establecen los valores donde comienza el bucle y variables que controlan las repeticiones.
Condición: esta elemento decide si el código se va a seguir repitiendo o no.
Actualización: el valor que se asigna a las variables que controlan el programa después de cada repetición.
Estructura do...while: esta estructura es muy parecida a la etructura while, lo que los hacen diferentes son, que en este bucle, el programa siempre se repite excepto la primera vez, esta estructura comprueba el programa cada vez que se repite, la primera vez siempre se ejecuta todas las instrucciones. En el while, siempre deben agregarse caracteres, esto es no se hace en el while común. Esta estructura se usa asi:
Do{
Instrucción
}while (condicion);

-MAP: es un mapa de clave/valor simple que permite iterar sus elementos en el orden en el que fueron insertados.Los objetos se han utilizado para asignar cadenas a valores, y permiten establecer claves, recuperar esos valores, eliminar claves y detectar si una clave almacena algo. Sus ventajas son:
+usar mapas sobre objetos hasta el momento de la ejecución cuando esta sea desconocida o cuando todas las claves y valores sean del mismo tipo.
+si es necesario usa mapas para almacenar valores primitivos (claves) por que el objeto trata a cada clave como una cadena, ya sea un valor numérico, booleano o primitivo.

    Se utilizan con una sencilla interfaz donde sus métodos son:
  • set( key, value ): añadir una nueva pareja clave-valor
  • +get( key ): obtener el valor asociado a una clave
  • delete( key ): borrar una pareja clave-valor por medio de la clave.
  • has( key ): comprobar si hay una determinada clave en el mapa.
Map tiene métodos y propiedades que son:
  • size: contiene el número de valores en el mapa.
  • forEach( (value, key, map) => {}, [thisValue] ): recorre todos elementos contenidos en el mapa.
  • values(): devuelve un iterable con los valores del mapa.
  • keys(): devuelve un iterable con las claves del mapa.
  • entries(): devuelve un iterable con matrices [key, value].
  • clear(): elimina todos los valores del mapa.
El map de convierte automáticamente en un iterable si se usa dentro de un bucle y el constructor de map puede aceptar una matriz o un itinerable- con un conjunto de pares clave-valor para inicializar el objeto. Puede ser utilizado como clave asociar valores a objetos, matrices, funciones, etc. Es un recurso útil para asociar información a cualquier elemento con el que se trabaje.
